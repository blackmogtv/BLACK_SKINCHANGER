# Lovable Prompt

Paste the prompt below into Lovable after connecting the app to Supabase.

```text
Build an internal admin dashboard for a Supabase-backed license system.

Design direction:
- Dark, premium control-panel UI similar to KeyAuth / the provided screenshot.
- Left sidebar with sections: Overview, Licenses, Products, Activity.
- Main Licenses page should feel dense and operational, not marketing-style.
- Use a charcoal background, subtle borders, green success accents, muted secondary text, compact cards, and monospaced license keys.
- Cards should look like admin records, not consumer product cards.

Authentication:
- Use Supabase Auth email login.
- After login, call the existing Supabase Edge Function `manage-keys` with the logged-in user JWT in the Authorization header.
- Do not expose any admin token in the client.
- If the user is unauthorized, show an "Access denied" state.

App requirements:
- Product selector at the top that loads products from `manage-keys` action `list_products`.
- Search input that filters license cards by key, note, HWID, or user label using `manage-keys` action `list_keys` with `search`.
- Grid/list toggle for license cards.
- License cards must show:
  - key at the top in monospace
  - status badge
  - created date
  - duration
  - generated by
  - used by
  - note
  - used on
  - HWID
- Actions menu on each card:
  - Copy key
  - Reset HWID
  - Ban / Unban
  - Edit note
  - Change duration
  - Delete
- Bulk select UI is optional, but single-item actions must work.

Required pages:
1. Login page using Supabase Auth email/password.
2. Licenses page:
   - product dropdown
   - search bar
   - create keys button
   - responsive grid of cards
   - details drawer or modal for one key with event history
3. Products page:
   - list products
   - create product modal
   - toggle active/inactive
4. Activity page:
   - show recent event history for a selected key when opened from the license details view

API contract:
- POST `/functions/v1/manage-keys`
- Use these actions:
  - `list_products`
  - `create_product`
  - `list_keys`
  - `get_key`
  - `create_keys`
  - `ban_key`
  - `unban_key`
  - `reset_hwid`
  - `update_note`
  - `set_duration`
  - `delete_key`
- Send `Authorization: Bearer <user access token>` in every request.
- Body is JSON.

Behavior details:
- On load, fetch products first and select the first active product by default.
- When product changes, refetch keys.
- Add debounce on search.
- Show optimistic UI for ban/unban, reset HWID, note updates, duration changes, and delete.
- For create keys, open a modal with quantity, duration in days or lifetime, generated by, and note.
- For change duration, allow null to represent lifetime.
- For delete, require a destructive confirmation modal.

Frontend stack:
- React + TypeScript
- Use Supabase client for Auth only
- Use fetch for Edge Function requests
- Strong typing for API responses
- Use reusable UI components and clean state handling

Visual details:
- Top breadcrumb bar
- Sidebar app switcher / product branding area
- Status badges:
  - USED = green
  - UNUSED = slate
  - BANNED = red
  - EXPIRED = amber
- Keys displayed in uppercase monospace with tight letter spacing
- Compact card spacing so many licenses fit on one screen
- Smooth hover states and crisp admin feel

Build the full UI with production-quality component structure, loading states, empty states, error states, and responsive behavior for desktop first.
```
